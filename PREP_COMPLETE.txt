DEPLOYMENT PATTERN IMPLEMENTED
================================================================================

All API calls now use this pattern:

    const apiUrl = process.env.NEXT_PUBLIC_API_URL || 'http://localhost:3000';
    const res = await fetch(`${apiUrl}/api/endpoint`, { ... });

Benefits:
  • Works with ngrok tunnel when NEXT_PUBLIC_API_URL is set
  • Falls back to localhost:3000 for local development
  • No hardcoded URLs
  • Easy to switch between environments

WHEN BRYDON SENDS NGROK URL
================================================================================

Brydon will provide:
  1. Docker running: docker-compose up --build
  2. ngrok running: ngrok http 3000
  3. ngrok URL: https://[something].ngrok.io

You run ONE command:
  ./DEPLOY.sh https://[ngrok-url-here]

The script handles:
  • Updating .env.local with ngrok URL
  • Building the Next.js project (npm run build)
  • Committing changes to git
  • Pushing to main branch

That's all. Everything else is pre-configured.
